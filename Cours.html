Cours GIT

Quest-ce que GIT?

Git est un outil de versionning, et il permet d'avoir une traçacbilité et un historique de tous les fichiers du projet.

Il permet également de travailler en équipe: Envoyer nos modifications à nos collègues et récuperer les modifications des collègues.

Dans la traçabilité des fichiers, GIT nous donne toujours 3 infos: Qui, Quand, Pourquoi?

Installer et configurer GIT

git config --global user-email 'n.hoarau75@gmail.com'
git config --global user-name 'nohdevam'

Les commandes de base

Initialaiser un projet:

git init

Voir le statut des fichiers:

git status

Ajouter des fichiers:

git add "nom du fichier"

Ajouter TOUS les fichiers:

git add .

commit (enregistrer) les fichiers ajoutés:

git commit
Un commit nécésiite toujours un message

commit avec un message:

git commit -m "message"

commit avec add + message (fonctionne sur les fichiers déjà add)

git commit -a -m "message"

Correction des fichiers

Voir les différents commit:

git log

Corriger les modifications sur un fichier (pas encore add):

git checkout 'nom du fichier'

Supprimer un add:

git reset 'nom du fichier'

Supprimer TOUS les add:

git reset . 'nom du fichier'

Supprimer le dernier commit:

git reset HEAD^

Supprimer 2 derniers commit:

git reset HEAD^^

La notion de branche

Un ebranche est une copie du projet. Elle permetde développer en parallèle de nouvelles fonctionnalités. Lorsqu'on considère les nouveautés opérationnelles on peut les rappatrier sur la version principale (master)

Voir les branches:

git branch

Créer une branche:

git branch 'nom'

Se placer sur une branche:

git checkout 'nom'

Voir les diff entre 2 branches:

git diff 'nom'..master

Apporter les modifs effectuées sur une branche dans master (il faut être ds master

git checkout master
git fetch .nom
le fetch va litteralement remplacer le master par la branche . Il s'agit donc d'une grosse modif ds le projet (ex: version 2)
